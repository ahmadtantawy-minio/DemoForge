## 2. Backend

### Language & Framework

Python 3.11+ with **FastAPI**. Key dependencies: `docker` (Docker SDK), `pydantic`, `pyyaml`, `jinja2`, `httpx`.

### Entry point — `backend/app/main.py`

### How API routes are registered

Each module in `backend/app/api/` exports a `router = APIRouter(prefix="/api/...")`. They are included via `app.include_router(...)`. The proxy router is last because it has a catch-all pattern.

### How the WebSocket / proxy system works

File: `backend/app/engine/proxy_gateway.py`

The backend joins each demo's Docker network at deploy time (via `network_manager.join_network()`). This gives it direct network access to all demo containers. Web UIs are proxied through:

```
/proxy/{demo_id}/{node_id}/{ui_name}/{subpath}
```

The proxy resolves the target container's hostname and port from the component manifest's `web_ui` entries:

```python
def resolve_target(demo_id: str, node_id: str, ui_name: str) -> tuple[str, str]:
    running = state.get_demo(demo_id)
    container_info = running.containers.get(node_id)
    manifest = get_component(container_info.component_id)

    for ui in manifest.web_ui:
        if ui.name == ui_name:
            base_url = f"http://{container_info.container_name}:{ui_def.port}"
            return base_url, ui_def.path
```

It uses `httpx.AsyncClient` for HTTP forwarding, rewrites `Location` headers, `Set-Cookie` paths, and injects `<base>` tags + fetch interceptors into HTML responses so SPAs work through the proxy.

Terminal access is via WebSocket at `/api/demos/{demoId}/instances/{nodeId}/terminal`.

```python
class ComponentManifest(BaseModel):

    variants: dict[str, VariantDef] = {}           # Named variants with command overrides
    template_mounts: list[TemplateMountDef] = []   # Jinja2 templates to render & mount
    static_mounts: list[StaticMountDef] = []       # Static files to bind-mount
    init_scripts: list[InitScriptDef] = []         # Post-deploy setup commands
```

**Required fields**: `id`, `name`, `category`, `image`.
**Everything else is optional** with sensible defaults.

### How variants work

Variants override the `command` field. The compose generator checks: if the node's variant exists in the manifest, use its command; otherwise fall back to the manifest's top-level command.

```python
variant = manifest.variants.get(node.variant)
command = variant.command if variant and variant.command else manifest.command
```

### How `web_ui`, `terminal`, and `health_check` are structured

- **`web_ui`**: Each entry has `name` (used in proxy URL), `port`, `path`, `description`. The proxy serves at `/proxy/{demo}/{node}/{name}/`.
- **`terminal`**: Defines `shell`, `welcome_message`, and `quick_actions` (label + command pairs shown in the terminal panel).
- **`health_check`**: `endpoint` + `port` -> translated to a Docker healthcheck using `curl -sf http://localhost:{port}{endpoint}`.

### How it reads manifests and produces services

```python
def generate_compose(demo: DemoDefinition, output_dir: str, components_dir: str = "./components") -> tuple[str, DemoDefinition]:
    demo = demo.model_copy(deep=True)       # Never mutate original
    project_name = f"demoforge-{demo.id}"

    # Build network map from demo.networks list
    network_map = {net.name: f"{project_name}-{net.name}" for net in demo.networks}

    # --- DemoCluster expansion (see below) ---
    # ... expands cluster definitions into synthetic nodes + edges ...

    services = {}
    compose_volumes = {}
    for node in demo.nodes:
        manifest = get_component(node.component)
        component_dir = os.path.join(components_dir, node.component)
        service_name = node.id
        container_name = f"{project_name}-{node.id}"

        # Variant -> command resolution
        if node.id in cluster_commands:
            command = cluster_commands[node.id]
        else:
            variant = manifest.variants.get(node.variant)
            command = variant.command if variant and variant.command else manifest.command

        # Merge environment: manifest defaults -> secrets resolution -> node.config overrides
        env = {}
        for key, val in manifest.environment.items():
            resolved = val
            for secret in manifest.secrets:
                placeholder = f"${{{secret.key}:-{secret.default}}}"
                if placeholder in val and secret.default:
                    resolved = secret.default
            env[key] = resolved
        env.update(node.config)

        # Build service dict
        service = {
            "image": manifest.image,
            "container_name": container_name,
            "expose": [str(p.container) for p in manifest.ports],
            "environment": env,
            "mem_limit": mem,
            "cpus": cpu,
            "labels": {
                "demoforge.demo": demo.id,
                "demoforge.node": node.id,
                "demoforge.component": manifest.id,
            },
            "networks": service_networks,
            "restart": "unless-stopped",
        }
        # ... add healthcheck, volumes, command, entrypoint ...
        services[service_name] = service
```

### How networking works (sandboxed proxy model)

Each demo gets its own Docker bridge network named `demoforge-{demo_id}-{network_name}` (typically `demoforge-{id}-default`). No ports are published to the host. Instead:

1. The backend container **joins each demo's Docker network** via `network_manager.join_network()`.
2. All container-to-container communication uses Docker DNS (e.g., `demoforge-lakehouse-minio-1:9000`).
3. The frontend accesses container web UIs through the backend's reverse proxy at `/proxy/{demo}/{node}/{ui}/`.

```python
# Network map generation
network_map = {net.name: f"{project_name}-{net.name}" for net in demo.networks}

# Each service joins networks with optional aliases and static IPs
service_networks = {}
for net_key in node_network_names:
    docker_net_name = network_map.get(net_key)
    net_entry = {}
    if net_cfg.ip:
        net_entry["ipv4_address"] = net_cfg.ip
    if net_cfg.aliases:
        net_entry["aliases"] = net_cfg.aliases
    service_networks[docker_net_name] = net_entry if net_entry else None
```

### Volumes

Named Docker volumes per node: `{project_name}-{node_id}-{volume_name}`. For cluster nodes with multiple drives, extra volumes are created: `{project_name}-{node_id}-data1`, `data2`, etc.

### Environment variables

Priority order (later overrides earlier):
1. Manifest `environment` defaults (with `${VAR:-default}` patterns resolved via `secrets`)
2. Cluster credentials (if node is part of a cluster)
3. Node-level `config` overrides from the demo definition
4. Auto-resolved S3 endpoints from edges (see Section 9)
5. Edge config mapped to env vars (e.g., `target_bucket` -> `S3_BUCKET`)

### Resource budget enforcement

```python
# Total demo budget — scale down per-container if total exceeds budget
if res.total_memory and _mem_bytes(res.total_memory) > 0:
    total_budget = _mem_bytes(res.total_memory)
    total_used = sum(_mem_bytes(s.get("mem_limit", "256m")) for s in services.values())
    if total_used > total_budget:
        scale = total_budget / total_used
        for svc in services.values():
            current = _mem_bytes(svc.get("mem_limit", "256m"))
            scaled = int(current * scale)
            svc["mem_limit"] = f"{max(scaled // (1024*1024), 64)}m"
```

## 8. Init Scripts / Post-Deploy Hooks

### File: `backend/app/engine/init_runner.py`

```python
async def run_init_scripts(demo: RunningDemo) -> list[dict]:
    # Group scripts by node_id, keeping intra-node order
    node_tasks: dict[str, tuple[str, list]] = {}
    for node_id, container in demo.containers.items():
        manifest = get_component(container.component_id)
        if not manifest or not manifest.init_scripts:
            continue
        scripts = sorted(manifest.init_scripts, key=lambda s: s.order)
        node_tasks[node_id] = (container.container_name, scripts)

    # Run each node's scripts in PARALLEL across nodes
    per_node_results = await asyncio.gather(
        *[_run_node_scripts(node_id, container_name, scripts)
          for node_id, (container_name, scripts) in node_tasks.items()]
    )
    return [r for node_result in per_node_results for r in node_result]


async def _run_node_scripts(node_id, container_name, scripts):
    results = []
    for script in scripts:
        if script.wait_for_healthy:
            healthy = await wait_for_healthy(container_name, script.timeout)
            if not healthy:
                results.append({
                    "node_id": node_id, "script": script.command,
                    "exit_code": -1, "stderr": "Timed out waiting for healthy"
                })
                continue
        exit_code, stdout, stderr = await exec_in_container(container_name, script.command)
        results.append({
            "node_id": node_id, "script": script.command,
            "exit_code": exit_code, "stdout": stdout, "stderr": stderr,
        })
    return results
```

### How it knows when dependencies are ready

Each `InitScriptDef` has `wait_for_healthy: bool` and `timeout: int`. If `wait_for_healthy` is true, the runner polls `get_container_health()` (Docker healthcheck API) until the container reports `HEALTHY` or times out.

```python
async def wait_for_healthy(container_name: str, timeout: int = 60) -> bool:
    elapsed = 0
    while elapsed < timeout:
        health = await get_container_health(container_name)
        if health == ContainerHealthStatus.HEALTHY:
            return True
        if health in (ContainerHealthStatus.ERROR, ContainerHealthStatus.STOPPED):
            return False
        await asyncio.sleep(2)
        elapsed += 2
    return False
```

### How init scripts are defined in the manifest

```yaml
init_scripts:
  - command: "sh -c 'until mc alias set local ...; do sleep 2; done && mc mb local/demo-bucket --ignore-existing'"
    wait_for_healthy: true
    timeout: 60
    order: 1          # Lower = runs first (within same node)
    description: "Create default buckets"
```

### Execution flow in deploy

Init scripts run **after** containers are started and discovered. Full deploy sequence in `docker_manager.py`:

```
generate_compose() -> docker compose up -d -> join networks -> discover containers
    -> run_init_scripts() -> generate_edge_scripts() -> auto-activate site-replication
    -> validate_and_reconcile()
```

Scripts within the same node run sequentially (ordered by `order`). Scripts across different nodes run in parallel via `asyncio.gather`.

### A. Environment variable injection (compose time)

### C. Edge automation scripts (post-deploy)

For runtime configuration (replication, tiering, site-replication), the `edge_automation.py` module generates shell commands executed inside containers after deploy. Each connection type has a registered generator:

```python
_GENERATORS: dict[str, callable] = {}

def _register(conn_type: str):
    def decorator(fn):
        _GENERATORS[conn_type] = fn
        return fn
    return decorator

def generate_edge_scripts(demo: DemoDefinition, project_name: str) -> list[EdgeInitScript]:
    scripts = []
    for edge in demo.edges:
        if not edge.auto_configure:
            continue
        generator = _GENERATORS.get(edge.connection_type)
        if generator:
            new_scripts = generator(edge, demo, project_name)
            scripts.extend(new_scripts)
    return sorted(scripts, key=lambda s: s.order)
```

- `cluster-site-replication` -> Cluster-level site replication
- `cluster-tiering` -> Cluster-level ILM tiering
- Template-driven (no runtime init): `metrics`, `metrics-query`, `file-push`, `s3`, `iceberg-catalog`, `sql-query`, `s3-queue`, `spark-submit`, `hdfs`, `failover`, `load-balance`

### Registry loader — `backend/app/registry/loader.py`

```python
_registry: dict[str, ComponentManifest] = {}

def load_registry(components_dir: str) -> dict[str, ComponentManifest]:
    global _registry
    _registry = {}
    for entry in os.listdir(components_dir):
        manifest_path = os.path.join(components_dir, entry, "manifest.yaml")
        if os.path.isfile(manifest_path):
            with open(manifest_path) as f:
                raw = yaml.safe_load(f)
            manifest = ComponentManifest(**raw)
            _registry[manifest.id] = manifest
    return _registry

def get_component(component_id: str) -> ComponentManifest | None:
    return _registry.get(component_id)
```
